## How to Use This Documentation
- Keep content concise and structured.
- Add new documents in the appropriate section.
- Use TODO sections when a decision is still open.
- Avoid implementation details that have not been formally decided.

## Future Documentation Guidance
Future documentation should be added in the most relevant area:
- product features go in [product](product)
- UI patterns go in [ui](ui)
- architecture topics go in [architecture](architecture)
- research notes go in [research](research)
- major product or technical decisions go in [decisions](decisions)
- release notes go in [releases](releases)
