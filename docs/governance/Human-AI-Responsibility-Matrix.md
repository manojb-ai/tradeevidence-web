# Human–AI Responsibility Matrix

**Status:** Approved  
**Version:** 1.0  
**Owner:** Chief Software Architect  
**Last Updated:** 2026-07-18
**Related Documents:** [AI-DLC-Adoption-Policy.md](AI-DLC-Adoption-Policy.md), [decisions/ADR-001-Adopt-AI-DLC.md](decisions/ADR-001-Adopt-AI-DLC.md), [../engineering/TradeEvidence-Engineering-Lifecycle.md](../engineering/TradeEvidence-Engineering-Lifecycle.md), [../08-AI-Strategy.md](../08-AI-Strategy.md)

| Activity | AI May Prepare or Execute | Human Accountability |
|---|---|---|
| Product intent | Questions, summaries, options | Founder approves |
| Scope | Draft boundaries and dependencies | Product owner approves |
| Architecture | Alternatives, trade-offs, diagrams | Chief Software Architect decides |
| Data model | Candidate schemas and validation | Architect/engineer approves |
| API design | Draft contracts and examples | API owner approves |
| UX behavior | States, edge cases, accessibility review | Product/UX owner approves |
| Code | Generate and refactor | Engineer reviews and owns |
| Tests | Generate cases and automate execution | Engineer verifies adequacy |
| Security | Threat analysis and scanner execution | Security owner accepts risk |
| AI behavior | Prompt and guardrail drafts | AI/product owner approves |
| Documentation | Draft and synchronize | Document owner verifies |
| Pull requests | Prepare changes and comments | Human approves merge |
| Deployment | Prepare plans and validation | Release owner authorizes |
| Production operations | Diagnose and recommend | Human authorizes material actions |
| Investment-facing language | Draft educational explanations | Human validates boundaries |

## Non-Delegable Decisions

Product direction, release authorization, acceptance of material security risk, production data deletion, user data access policy, scoring-model approval, AI recommendation boundaries, and legal/compliance representations remain human decisions.
