# 02. Principles

These principles define how TradeEvidence should behave as a product and as a platform.

## Core Principles

| Principle | Meaning |
| --- | --- |
| Evidence before emotion | The product should support reasoning grounded in evidence rather than impulse or hype. |
| AI assists; the trader decides | AI can support analysis and summarization, but the trader remains responsible for the final decision. |
| Every score should be explainable | Scores must be traceable to visible factors and evidence. |
| Never hide assumptions | Assumptions, limits, and data quality should be visible. |
| Transparency over black-box predictions | The product should favor interpretable logic over opaque outputs. |
| Professional over flashy | The experience should feel calm, credible, and focused on work. |
| Consistency over complexity | Simplicity and repeatable workflows are more valuable than unnecessary feature depth. |
| Show confidence and uncertainty | The product should distinguish between strong evidence and uncertain context. |
| Make disciplined trading easier | Tools should reduce friction in research, review, and journaling. |
| Every feature should improve decision making | New capabilities should support better judgment, not distract from it. |

## Behavioral Commitments
- The trader owns the final decision.
- The platform should encourage reflection, not automation of judgment.
- Clear context should be available for every score, alert, or insight.
- The product should support disciplined execution with transparent reasoning.

## Product Tone
TradeEvidence should feel analytical, professional, and trustworthy. It should help users think clearly, not tell them what to do.
