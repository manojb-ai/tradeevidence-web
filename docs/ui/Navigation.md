# Navigation

## Purpose
Navigation should make it easy to move between research, analysis, journaling, and settings.

## Users
- traders working across modules
- users returning to prior workflows

## Features
- primary module navigation
- fast access to recent contexts
- clear information hierarchy

## Future ideas
- workspace switching
- saved views
- context-based shortcuts

## Dependencies
- product information architecture
- dashboard and module requirements

## Open Questions
- What is the minimum navigation model for v0.3?
- Should navigation be persistent or context-sensitive?
